Introduction

The main purpose of this practical is to give us a basic intro to direction dependent (DD) calibration as implemented in MeqTrees. If you recall from previous lectures and MeqTrees practical, that 2GC methods assume that all propagation effects are direction independent (DI) and any present DD effect is the same across all the receivers (trivial DDEs). This approximation is valid for narrower field of views but has already started showing its limitations even for smaller arrays such as the KAT7. The DD calibration technique implemented on MeqTrees is called the differential gains (DGs). This approach solves for G-Jones (DI) and DD effects simultaneously. It does this by estimating and correcting for receiver gain drifts (i.e GJones) on short time/frequency scales and the DGs on longer time/frequency scales. So the G solutions subsume all the DDEs towards the dominant source in the field and the DGs solves for these DDEs in the direction of these sources.

In this practical, we are going to use real observation data whereas in the previous MeqTrees 2GC practical we had to generate our own visibilities (through simulations) to calibrate. We going to calibrate the KAT7 data.

Imaging the Field

Let us start by imaging the visibilities so that we can have some idea of what we are in for. Open meqbroswer, press start, go to TDL then Load TDL Script and navigate to Cattery then calico and load calico-stefcal.py. On the open "TDL Compile-time Options" Load the MS file and click Compile . A "TDL Jobs & Runtime Option" window should appear and expand the "Imaging Options" and set the following;
  • Image type or column: DATA
  • Name of output FITS file: name this Original_field_clean.fits
  • Image size, in pixels: 1024
  • Image size, in arcmin: 180
  • Expand the Make a clean Image
  • Clean Algorithm: clark . Fortunately, this clean algorithm can make a clean image from the DATA column.
  • Number of iterations: 3000
  • Leave the rest as defaults
  • Click on Make a clean image
You should see the image below. I have used Tigger to change the dynamic range of the image. You can also changing this by click on the red, yellow, green bars at the bottom write of the open tigger image and cliping the noise distribution plot that appears or by simply editing the numbers at the bottom. I used [-0.001958, 0.006591].
